HTML Elements and HTML Tags

Before you start designing and coding your first web site we need to first explore HTML Elements and HTML Tags in more detail.

  • Recall that HTML source code is plain text so we can use any text editor to create or modify HTML source code to create a web page. HTML is a mark up language, meaning it is used to mark up content in a document. In fact HTML stands for HyperText Markup Language. HyperText refers to the linking between documents (i.e. the links on a web page). The markups in HTML specify how the content should be interpreted and rendered by a web browser; this web page (and all the web pages on the website) are written in HTML.
    • If you right-click on a webpage, you can select "View page source" and your browser will show you the HTML source code of the page.

  • HTML elements are the building blocks of HTML pages. Different elements specify different types of content within an HTML document. HTML elements are represented by HTML tags. HTML tags define what type of element it is and labels pieces of content such as "paragraph", "boldface", "lists", etc. Before we see how different tags are rendered, we first need to discuss the syntax of HTML Elements and HTML Tags.

The below diagram illustrates the basic HTML terms. Definitions of the terms follow the diagram. In this case an "anchor" element (defined by the <a> tags) is used in the middle of a line of text that will appear on the webpage. The anchor element is used to place a clickable link on a webpage. Note that the "I like to shop at" and "in Joesville" portions are not part of the element, but will show up on the webpage as plain text.

HTML Element
The block of HTML source code that includes the tags, and content between the tags.
Most HTML Elements include a start tag and an end tag, there are a few HTML Elements that only include a start tag.
Syntax: Start-Tag Inner-HTML End-Tag
E.g. <A href="https://www.google.com">LMGTFY</A>
HTML Tag
Used to mark (identify) content within an HTML document.
HTML uses a Start Tag to mark the beginning of the content, and an End Tag to mark the end of the content.
HTML Tags follow a specific syntax, start tags and end tags have a similar but different syntax.
Angle brackets (<, >) to indicate the beginning and end of an HTML Tag.
HTML tags can be uppercase or lowercase, although lowercase is preferable.
A < is used to indicate the beginning (opening) of an HTML Tag.
A > is used to indicate the end (closing) of an HTML Tag.
Start Tag
Marks the start of an HTML Element.
Start Tags may contain amplifying information about the marked content using attributes.
The element type follows the tag opening, <, the tag type indicates what kind of element the HTML Element is.
Attributes follow the element type.
Syntax: <ELEMENT-TYPE list-of-attributes>
E.g. <A href="https://www.google.com">
End Tag
Marks the end of an HTML Element.
Does not contain attributes.
A / precedes the element type after the tag opening, <.
An end tag is paired with a start tag.
Syntax: </ELEMENT-TYPE>
E.g. </A>
Inner HTML
The inner HTML is the content that is being marked by the start and end tags and is the part that will be rendered on the webpage. It is located in between the start tag and the end tag.
Inner HTML can contain other elements; that is HTML Elements can be nested in the inner HTML or a parent HTML Element.
Syntax: (Mix of plain text content and/or other HTML Elements)
E.g. LMGTFY
Attribute
Used to specify amplifying information associated with the marked inner HTML. In other words, they can apply additional affects to the innerHTML (the content) beyond that specified by the base tags. Examples include changing the color, size, and/or font of a piece of text.
Some attributes can be used in any HTML Element, other attributes are specific to certain HTML Elements.
Attributes appear in a start tag, valid HTML does not have attributes in end tags.
Most attributes are comprised of name-value pairs, there a a few attributes that are just a name.
An equal sign (=) is used between an attribute name and the attribute value.
You can think of a name-value pair as assigning the value to the attribute named name.
Syntax: attribute-name=value
E.g. href="https://www.google.com"
Attribute Name
Specifies the name of the attribute being set.
Appears to the left of the equal sign if a value is being assigned to the attribute.
Attribute names are a part of the HTML specification, the name of the attribute means something in HTML.
E.g. href
Attribute Value
The value being assigned to the attribute.
Appears to the right of the equal sign.
The value is interpreted based on attribute name, sometimes the value may be interpreted as a URL, sometimes as a value from a set.
E.g. "https://www.google.com"

HTML Tags

Now that you know the basic terms of HTML elements, let's look at some of the basic HTML tags and attributes that you will be using.

Structural tags:

  • <!DOCTYPE>
    Although this looks like an HTML tag is is not. This is actually an instruction to the web browser about what version of HTML the page is written in. This must be the first thing in your HTML document, before the <html> tag. Although not strictly required, it helps your web page load better as the web browser knows what version of HTML to expect. This course will use HTML 5.

  • <html>...</html>
    Tells the browser that this is an HTML document and is the root of an HTML document. The HTML tag is the container for all other HTML elements.

  • <head>...</head>
    This is the container for the head elements. It can include the title for the document, overall style information, and other information. You'll primarily use this for the title.

  • <title>...</title>
    The <title> tag is should be used in all HTML documents. It defines the title of the document. The title is what shows up on the tab for the page at the top of the browser window. It also provides a name for the page when it is added to favorites and displays a title for the page in search-engine results.

  • <body>...</body>
    The body element contains all the elements of an HTML document, such as text, hyperlinks, images, tables, lists, etc that you want to be rendered on your web page. Most of the content of your web page will be contained within the body element.

The structural tags above form the basic layout of an html file (a web page) as shown below: 
<!DOCTYPE html>
<HTML>
  <HEAD>
    <TITLE>Title of Web Page</TITLE>
  </HEAD>
  <BODY>
    Content of Web Page: Headings,
    Paragraphs, Links, Images, etc.
  </BODY>
</HTML>

Format and content tags

  • <h1>...</h1> : heading
    The heading tag formats content as a chapter heading. They go from <h1> to <h6>. <h1> denotes the most important heading and <h6> denotes the least important heading. Headings are bold, form their own paragraph, and vary in size from <h1> being biggest to <h6> being smallest.

  • <b>...</b> : Bold, formats text as bold.

  • <u>...</u> : Underline, formats text as underlined

  • <i>...</i> : Italics, formats text in italics.

  • <p>...</p> : paragraph
    The paragraph tag formats the content as its own paragraph. With HTML, if you want certain content to be together in a paragraph, you have to tell the browser to do so. Just because you put something on a new line in the html file, does not mean it will be rendered by a browser that way. Browsers automatically add some space before and after each <p> element.

  • <br> : break
    Sometimes you may not want to group content into a paragraph, but just want a new line. The <br> tag functions like the "enter" key in a text editor in that it creates a single line break. Note that this is one of the HTML tags that does not have an end tag or any content; called an "empty tag".

  • <a>...</a> : anchor
    The <a> tag is used to define a hyperlink (link) that when clicked, takes your browser to a different web page. The <a> element uses the href attribute to define the URL of the web page the link will take your browser to. The content in between the start and end tags is what is displayed on the webpage as the link. An unvisited link is underlined and blue. A visited link is underlined and purple.

  • <img> : image
    The <img> tag is used to define an image in a web page. The <img> tag is another empty tag in that it does not have an end tag. The src attribute is required for this tag and tells the browser which image file to use and display on the web page. As your browser reads the HTML and renders the web page, if it reaches an <img> tag, it requests the associated image file from the URL specified by the src attribute then renders the image on the web page.

  • <span>...</span> : span
    The <span> tag does not alter text by itself, but provides a way to apply attributes to text. There may be times where you want to apply an attribute to, for example, change the color of a word but do not want the effects provided by another tag such as <b> to take effect. The <span> tag allows you to do this.

  • Lists
    • You can create lists with HTML. Lists use the <li>...</li> tags for each item in the list.
    • <ul>...</ul> : unordered list
      The <ul> tag is used to designate an unordered list. Each item in the list will be bulleted.

    • <ol>...</ol> : ordered list
      The <ul> tag is used to designate an ordered list. Each item in the list will be numbered.

Attributes

All HTML elements can have attributes. Attributes provide additional information about an element and are always specified in the start tag. Attributes come in name/value pairs in the format name="value".
  • src
    • The src attribute is often used with the <img> tag to specify the filename of the image to be rendered on the web page. It is formatted such that src is the attribute name and the filepath/filename for the image is the attribute value. The filepath/filename can be either a full URL or a relative filepath for a file located on the same server as the current web page html file. For example:

    • Full URL source (absolute reference) example: <img src="http://www.foxtrot.com/wp-content/uploads/2014/07/ft111002noncompliant.png">
    • Relative reference example: <img src="SleepyFace.JPG">

  • href
    • The href attribute is used with the <a> tag to specify the URL that the link should go to when clicked. The filepath/filename can be either a full URL or a relative filepath for a file located on the same server as the current web page html file. For example:

    • Full URL (absolute reference) example: <a href="https://www.w3schools.com/html/html_intro.asp">Introduction to HTML</a>
    • Relative reference example: <a href="interests.html">A relative link to your interests page</a>

  • style
    • The style attribute can be used to add style properties to the content of an element such as background color, text color, fonts, text size, etc. More style information

  • width and height are two attributes that allow you to scale content like images to the size you want it to be on the web page. You can specify a specific size in pixels or you can also use height and width values that allow the content to scale with the size of the browser window.
    • Size in pixels example: <img src="SleepyFace.JPG" width="500px" height="600px">
    • Automatic scaling example: <img src="SleepyFace.JPG" width="100%" height="auto">

  • id
    • The id attribute is a special attribute that allows you to uniquely identify a specific element by assigning it a name (id).
